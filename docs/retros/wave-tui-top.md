## Что сработало хорошо

1. **Реюз вместо дубля**: вся сборка данных — существующий collectors.mjs (не тронут,
   19 self-tests зелёные). Новый код — только отрисовка + 1 поле. Картограф до спеки
   подтвердил отсутствие дублей — нулевая переделка.
2. **TDD с исполняемым харнессом**: красная фаза зафиксирована реальным прогоном до кода,
   definition of done у строителя был механическим (exit 0), не словесным.
3. **Конфликт брифов разрешён ADR-ом в спеке** (Bubbletea vs zero-dep): строитель не гадал.
4. **Reflexion-critic поймал реальный изъян** (терминал «киприливается» при крэше панели:
   alt-screen + скрытый курсор + raw mode без восстановления). Исправлено в round 2
   с unit-доказательством (AC-crash: restore() эмитит \x1b[?1049l на crash-пути).

## Уроки (для будущих TUI/CLI волн)

1. **Любой alternate-screen TUI обязан регистрировать restore() на process.on('exit') +
   uncaughtException + unhandledRejection, не только на q/SIGINT/SIGTERM.** Иначе первый
   же необработанный throw в render-цикле оставляет пользователя в пустом терминале без
   курсора и без эха. Это требование уровня спеки, не «доработка потом».
2. **Субагент без shell-доступа не может выполнить "запусти и покажи вывод"** — test-engineer
   честно отдал трассировку вместо прогона; оркестратор обязан перепроверять прогоном сам
   (правило «не верь на слово» сработало: трассировка имела 1 ошибку — AC-2, агент сам поправил).
3. **Сводки агентов могут расходиться с реальностью в мелочах** (например «14 кейсов
   collectors» при реальных 19) — числа в промптах задач сверять с кодом, не с пересказом.
4. **terminalId из env — случайный UUID сессии, не PII** (проверено конституционным ревизором),
   но локальные Kaizen-логи (`docs/audits/tui-panel-launches.jsonl`) — сразу в .gitignore.

## Kaizen-петля

Метрика: время до обнаружения застрявшей волны (цель ≤5с от открытия панели).
Измерение: docs/audits/tui-panel-launches.jsonl ({ts, project, wavesInFlight, stuckCount}) —
частота запусков покажет, пользуются ли панелью; корреляция запуска с первым --advance
застрявшей волны покажет время реакции. Читать на следующей kaizen-итерации (data-analyst).
