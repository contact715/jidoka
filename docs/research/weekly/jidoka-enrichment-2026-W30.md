## 3. Что усилить из сильных сторон

- **Инвариант «ноль зависимостей, чистый Node» — несущий фильтр.** В этом раунде он снова отсёк ПАКЕТ у 6 из 11 источников (OpenHands, claude-flow, disler, humanlayer, LoCoMo и пакеты graphiti/gepa/A-mem/spec-kit), пропустив только ~40-строчную нативную идею. Держать его первым несменяемым гейтом на каждое предложение.
- **«Механизмы, а не декларации» самопроверяется на своём же репозитории:** instantiation-audit ловит 0 призраков, gate-audit находит гейты без входящих ссылок. Любой новый скрипт этой недели обязан приземлиться с живым вызывающим, иначе будет пойман как orphan.
- **Замкнутая петля мета-честности поймала свою же неточность** в этом прогоне (расхождение LEARNING/REGRESSING и неверные checkPaths) и сама исправила до синтеза. Усилить, привязав checkPaths аудитора к реальным путям (см. быстрые победы).
- **Детерминированный eval-suite 100%, регрессия рушит CI.** Каждый новый механизм ниже блокируется регресс-фикстурой ровно как существующие.
- **Честная самодокументация (DORMANT/SKIP вместо фальшивого зелёного).** Единственное место, где эта сила гниёт — устаревшие honest-state доки; поправить быстрой победой.

### Ранг 1 — P1 — Нативный event-log + node-events поток (строим W29-R6, пакет НЕ тянем)
- **Источник:** LangGraph (langchain-ai/langgraph), сходится с OpenHands SDK и humanlayer factor-12. https://github.com/langchain-ai/langgraph
- **Что:** ~40 строк чистого Node — append-only JSONL-чекпоинт после каждого успешного узла DAG, по ключу wave_id (`{wave, node, agent, status, outputHash, ts}`). Состояние = reduce(events). Три независимых источника сошлись на этой же примитиве — сильный сигнал, что это правильный нативный порт.
- **Киллер-свойство:** одна примитива закрывает три стоящих пункта сразу — (1) детерминированный crash-resume повтором вместо эвристической реконструкции с диска, (2) человек-в-цикле через узел, паркующийся на `status:'interrupt'`, (3) недостающий поток временных меток, оживляющий `compute-dora`/`compute-slos` lead-time.
- **Какой пробел закрывает:** мёртвая observability; нет crash-resume (умерший mid-DAG заново диспатчит готовые узлы); dag-schedule оптимизирует непроверенный критический путь.
- **Точка встройки:** новый `scripts/checkpoint.mjs` (append + loadCompleted, терпим к обрезанной последней строке); `scheduleDAG()` в `dag-schedule.mjs` получает опциональный `completed` и фильтрует готовые узлы; цикл диспатча в `orchestration-planner.mjs` зовёт `checkpoint.record()` и паркуется на interrupt; телеметрия со-эмитит через существующий `emit-telemetry.mjs`, который уже читает `andon-resume.mjs`. Все точки касания — живые вызывающие, риска orphan нет.
- **Усилие:** среднее (~40 строк ядро + подключение; половину с interrupt ОБЯЗАТЕЛЬНО доказать исполняемым self-test «убей mid-DAG → resume пропускает готовые узлы, interrupt-узел блокирует», иначе это регресс в собственный класс движка core-property-substituted-by-scaffold).
- **Вердикт дебатов:** ADAPT (пакет LangChain отклонён по инварианту, портируется только keyed-checkpoint).
- **Новое на неделе:** нет (это нативный W29-R6, поднятый с P2 в keystone).

### Ранг 2 — P1 — Contradiction → INVALIDATE: четвёртый вердикт memory-guard (новое)
- **Источник:** getzep/graphiti (~28 972 звёзд, проверено gh api 2026-07-20). https://github.com/getzep/graphiti
- **Что:** добавить четвёртый исход в `memory-guard.judgeMemoryWrite`: когда новичок разделяет субъект существующего урока, но переворачивает его полярность (консервативное детектирование negation-cue + общий субъект), вернуть `{admit:true, invalidates:<id>}` и штампануть СТАРЫЙ урок существующим маркером `[superseded YYYY-MM-DD by: …]`. Порт только би-темпоральной идеи; LLM-экстракция, Neo4j и эмбеддер graphiti остаются за бортом.
- **Киллер-свойство:** закрывает интервал валидности факта вместо того, чтобы перевёрнутый урок сосуществовал и всплывал вечно. Потребитель УЖЕ живой — `memory-consolidate.mjs` уже парсит SUPERSEDE_MARKER и уводит помеченные кластеры в хвост «История» — поэтому как только гейт штампует маркер, устаревший урок перестаёт всплывать. Ни нового потребителя, ни эмбеддера, ни человеческой разметки — полностью обходит блокер калибровки судей.
- **Какой пробел закрывает:** у memory-guard только три shared-исхода (MERGE-суперсет / near-dup BLOCK / admit) без исхода «противоречие», поэтому перевёрнутый урок сохраняется в полном весе и, так как consolidate ранжирует по свежести-частоте, устаревший всплывает вечно. `memory-supersede-check.mjs` советующий, вне пути записи и слеп к прозаическим переворотам.
- **Точка встройки:** `memory-guard.mjs`, внутри блока `verdict === 'shared'`, ПЕРЕД суперсет-MERGE-циклом (противоречие обязано перебивать merge): консервативный negation-классификатор + ветка INVALIDATE + self-test-фикстура. В `memory-consolidate.mjs` менять ничего не надо. Зеркалить в `~/.claude/jidoka/scripts`.
- **Усилие:** низкое (~40 строк детерминированной строковой логики; классификатор держать консервативным против ложной инвалидации).
- **Вердикт дебатов:** ADAPT (высокая уверенность — пакет отклонён, одна идея портирована).
- **Новое на неделе:** да.

### Ранг 4 — P2 — Ретроактивное обновление соседей в memory-consolidate (детерминированный class-граф) (новое)
- **Источник:** agiresearch/A-mem. https://github.com/agiresearch/A-mem
- **Что:** два чистых прохода в `consolidate()` после кластеризации и до рендера: `buildLinks(clusters)` пишет `links[]`/`prevInClass[]` на кластер (закрывает открытый W28-R7), а `evolve(clusters, newRows)` пересчитывает окрестность связей у СТАРЫХ родственных кластеров при приходе новых строк леджера. Связи — только детерминированные диск-сигналы: kebab-префикс класса-семьи, пересечение `projects[]`, со-встречаемость в одной волне.
- **Киллер-свойство:** превращает изолированный-по-точному-классу дайджест в ассоциативный, где родственные семьи ошибок всплывают вместе (например core-property-substituted-by-scaffold ↔ reactive-literal-execution) — доставляя ценность, обещанную memory-vector, но БЕЗ отсутствующего эмбеддера и БЕЗ LLM, обходя оба блокера. Это ещё и граф-субстрат, который банкованному W29-R7 PPR-реранкеру физически нужен.
- **Какой пробел закрывает:** `memory-consolidate` — чистый scan→groupByClass→decay→render без рёбер заметка-заметка; ретроактивного обновления старых заметок при новом приходе нет нигде.
- **Точка встройки:** `memory-consolidate.mjs` внутри `consolidate(rows, today, remedies)`; `render()` печатает строку «см. также →» рядом с каждым уроком (потребляется в ту же сессию — consolidate идёт на старте каждой сессии и в kaizen-loop). Гейт: фикстура-леджер + ожидаемое множество рёбер как новый eval-кейс (точное равенство множеств); instantiation-audit подтверждает, что каждая связь указывает на реальный кластер. Явная граница: НЕТ ChromaDB, НЕТ sentence-transformers, НЕТ LLM.
- **Усилие:** низко-среднее (~40 строк детерминированно).
- **Вердикт дебатов:** ADAPT (средне-высокая — пакет отклонён, ретро-обновление — по-настоящему новая детерминированная способность, апгрейдит тонкий стаб W28-R7).
- **Новое на неделе:** да (половина buildLinks закрывает W28-R7; половина evolve — новая).

### Ранг 5 — P2 — Pareto-поиск промптов над детерминированными match-векторами голден-кейсов (новое)
- **Источник:** gepa-ai/gepa. https://github.com/gepa-ai/gepa
- **Что:** ~40-строчный `paretoArchive()` над per-case match-векторами из `score()` в `llm-eval-score.mjs`, дополняющий строгий скалярный no-regression accept-гейт в `prompt-evolution.mjs`. Держать Pareto-фронт вариантов промпта (каждый лучший на своём подмножестве голден-кейсов) вместо требования одной монотонной победы без регрессий.
- **Киллер-свойство:** ВТОРОЙ движок самоулучшения на УЖЕ детерминированном, без-судейском категориальном match-сигнале (`llm-eval-score` не зовёт ни LLM, ни судью, ни fetch — проверено), поэтому полностью обходит замороженный P0 калибровки судей и выходит из ловушки локального оптимума «сломал другое — отклонено» у prompt-evolution.
- **Какой пробел закрывает:** `prompt-evolution.isImprovement()` — строгий скалярный accept-гейт локального оптимума (grep pareto по scripts+agents = 0); мульти-кандидатного кросс-ранового поисковика нет, и он не дублирует зрелый адверсариальный стек.
- **Точка встройки:** `prompt-evolution.mjs`: `paretoArchive(candidates)` над `results[].match` из `llm-eval-score.mjs` (~10 измеренных агентов в `docs/evals/*`). Предлагатель = существующий `~/.claude/agents/prompt-evolver.md`. Ship-гейт остаётся `isImprovement()`+человек — Pareto только ИЩЕТ/архивирует, никогда не авто-применяет (композится с anti-reward-hacking). Добавить `--self-test`.
- **Усилие:** средне-высокое — архив ~40 строк, но ОБЯЗАТЕЛЬНЫЙ со-реквизит и есть реальная цена: автоматизировать цикл генерации кандидатов (предложить → пере-диспатчить агента на его голден-набор → пере-оценить) И вырастить голден-наборы за пределы 2-6 кейсов, иначе фронт вырожден и архив спит. Это условие ADAPT.
- **Вердикт дебатов:** ADAPT (условно — пакет отклонён, нативная идея принята ТОЛЬКО в связке с автоматизацией питающего её цикла кандидатов).
- **Новое на неделе:** да.

## 7. КИЛЛЕР-ФИЧИ НЕДЕЛИ

Рычаги, меняющие КАК мы работаем, а не «ещё один скрипт».

### Киллер-фича A — Нативный event-log как keystone инфраструктуры (Ранг 1)
- **Что:** одна append-only примитива чекпоинтов, из которой разом вытекают crash-resume, человек-в-цикле и оживление observability.
- **Почему рычаг, а не скрипт:** три независимых источника (LangGraph, OpenHands, humanlayer) сошлись на ней; она разбанковывает W29-R9 и питает детекцию scope-escape; даёт данные, ради которых вообще строился DAG-планировщик. Меняет движок с «планирую вслепую» на «вижу и переигрываю по факту».
- **Где живёт:** движок jidoka (`scripts/checkpoint.mjs` + подключение в `dag-schedule.mjs`/`orchestration-planner.mjs`), зеркало в `~/.claude/jidoka`.
- **Какой класс боли убирает навсегда:** «умерла волна mid-DAG — переделываем готовое»; «lead-time физически неизмерим».
- **Усилие/приоритет:** среднее / P1 (с обязательным kill-resume self-test).

### Киллер-фича B — Разблокировка калибровки судей через PPI/PPI++ вместо Rogan-Gladen (стратегический сдвиг)
- **Что:** когда наконец строим калибровку (W28-R1/W29-R2), делать оценщик на PPI/PPI++, а не на ванильном Rogan-Gladen. Литература 2026 (arXiv 2601.05420, 2606.05308) показывает, что PPI доминирует по конечно-выборочному смещению и ширине доверительного интервала при том же ~40-строчном усилии, требуя МЕНЬШЕ человеческих меток, чтобы объявить судью «измеренным».
- **Почему рычаг, а не скрипт:** меньше человеческих меток — это ровно тот бутылочный горлышко, что держит P0 замороженным четыре отчёта. Смена оценщика меняет саму достижимость всей замороженной половины дорожной карты памяти (reasoning-distill Part B, стратегии-инъекции).
- **Где живёт:** движок jidoka (`judge-calibration.mjs`), плюс запись в память как урок «PPI > Rogan-Gladen для калибровки на малой разметке».
- **Какой класс боли убирает навсегда:** «рекомендация open четыре недели, потому что нужна недостижимая ручная разметка».
- **Усилие/приоритет:** крупная ставка / P0 по рычагу (но зависит от построения W29-R2 голден-набора первым).

### Киллер-фича C — Второй движок самоулучшения на без-судейском сигнале (Ранг 5)
- **Что:** Pareto-архив над детерминированными match-векторами голден-кейсов — самоулучшение промптов агентов, которое работает СЕЙЧАС, пока судьи заморожены.
- **Почему рычаг:** сегодня всё самоулучшение агентов упирается в один заблокированный P0. Второй движок на уже-детерминированном сигнале снимает единственную точку отказа и выводит из локального оптимума prompt-evolution.
- **Где живёт:** движок jidoka (`prompt-evolution.mjs`) + агент `prompt-evolver.md` в `~/.claude`.
- **Усилие/приоритет:** средне-высокое / P2 (только со связкой автоматизации цикла кандидатов).
