## STRIDE Categories

### S — Spoofing

**Definition**: An attacker impersonates a legitimate entity (user, service, agent, data source).

**Trigger questions — this project context:**

1. Can a caller claim to be a different this project tenant when calling an `/api/` endpoint? What validates identity beyond a JWT?
2. If a webhook arrives claiming to be from Meta/Twilio, what prevents a spoofed payload from triggering lead creation or status changes?
3. Can a user impersonate an agent (e.g., submit a response as if it were generated by an LLM agent) through the conversation API?
4. Could a compromised session token allow one tenant to issue requests as another? What is the tenant isolation enforcement point?

**Relevant LLM threats**: LLM07 (System Prompt Leakage — attacker learns prompt to better impersonate system behavior).

---

### T — Tampering

**Definition**: An attacker modifies data in transit or at rest without authorization.

**Trigger questions — this project context:**

1. Are agent instruction files (`.claude/agents/*.md`, `docs/quality/*.md`) writable by any tenant-level user? What prevents a tenant from altering criteria to lower gate thresholds?
2. Can a lead record's status be changed via a CSRF attack on the dashboard mutation endpoint? Is `SameSite=Strict` or CSRF token enforced?
3. If `perf-budget.json` or `wcag-2.2-checklist.md` is modified after a BLOCK verdict, is there an audit trail? Does the PFCA log include a hash of the criteria doc read?
4. Can a WhatsApp/Meta message payload be tampered in transit (replay attack, body modification) before it writes to the message store?

**Relevant LLM threats**: LLM01 (Prompt Injection — malicious content in lead messages could tamper with agent instructions).

---

### R — Repudiation

**Definition**: A party denies having performed an action, and there is no audit evidence to contradict the claim.

**Trigger questions — this project context:**

1. When a lead status is changed by an automated agent vs. a human operator, is the actor clearly logged and non-repudiable?
2. If an LLM agent sends a WhatsApp message to a lead, is the sending event logged with agent identity, wave version, and timestamp — not just the message content?
3. When a WAIVED entry is added to `§13 Cross-Cutting Gates`, is the waiver author and timestamp captured in the spec's git history? Is this sufficient as audit evidence?
4. Is there a non-repudiable record of which agent version produced each BLOCK or PASS verdict in `docs/audits/checklist-runs.jsonl`?

**Relevant LLM threats**: LLM06 (Excessive Agency — agent takes actions the human cannot audit or dispute).

---

### I — Information Disclosure

**Definition**: Data is exposed to unauthorized parties.

**Trigger questions — this project context:**

1. Does the STRIDE template itself (this file) expose internal data flow topology or security control details if the repo becomes public? What is the repo visibility policy?
2. Can a tenant access another tenant's lead data, conversation history, or knowledge base entries through IDOR on any `/api/leads`, `/api/messages`, or `/api/knowledge` endpoint?
3. Does the LLM system prompt ever include sensitive customer data (PII, payment info)? If so, what prevents LLM07 system prompt leakage?
4. Are stack traces or internal error messages exposed in API error responses to the frontend? Verify `NODE_ENV=production` strips debug output.

**Relevant LLM threats**: LLM02 (Sensitive Information Disclosure), LLM07 (System Prompt Leakage).

---

### D — Denial of Service

**Definition**: An attacker disrupts service availability for legitimate users.

**Trigger questions — this project context:**

1. The `@axe-core/playwright` scan step runs on CI. Can a malformed page component cause the scan to hang indefinitely? Is there a `--timeout` flag applied?
2. Can a high-volume inbound webhook flood (Meta/Twilio) exhaust the FastAPI worker pool? Is rate limiting or queue-backed processing in place?
3. If the LLM provider (OpenAI Realtime, Gemini) becomes unavailable, does the conversation surface degrade gracefully or does it block the entire dashboard?
4. Can an LLM agent in an unbounded loop consume excessive tokens, causing cost-based DoS (LLM10)? Is per-request token budget enforced?

**Relevant LLM threats**: LLM10 (Unbounded Consumption — runaway agent token usage).

---

### E — Elevation of Privilege

**Definition**: A user gains capabilities beyond their authorized role.

**Trigger questions — this project context:**

1. Can a technician-role user access admin-only endpoints (`/api/company`, `/api/users` admin actions) through direct URL manipulation? Is RBAC enforced at the API layer, not just the frontend?
2. Can a WAIVED entry in `§13 Cross-Cutting Gates` be added by any agent without requiring human approval? Is the PFCA check gated on human review for WAIVED entries?
3. Does the Google OAuth flow prevent an unregistered email from gaining access to any tenant's data if they have a valid Google token?
4. If a malicious prompt injection (LLM01) causes an agent to invoke privileged API actions outside its declared scope, what prevents privilege escalation beyond the agent's F4 decision rights?

**Relevant LLM threats**: LLM01 (Prompt Injection leading to privilege escalation), LLM06 (Excessive Agency — agent acts beyond declared decision rights).

## this project LLM Exposure Map (OWASP LLM Top 10 2025)

| LLM Risk | Rating | Surface | Mitigation |
|---|---|---|---|
| **LLM01 Prompt Injection** | HIGH | Inbound lead messages fed to conversation agent; knowledge base content; Meta webhook payloads | `scripts/detect-injection.mjs` OWASP LLM01:2025 pattern battery (wave-165 runtime enforcement); sanitize user-generated input before including in prompt; agent decision rights cap (F4/ADR-019); never execute tool calls from untrusted content |
| **LLM02 Sensitive Information Disclosure** | HIGH | System prompt may include customer PII; conversation history stored in DB | PII masking before prompt injection; system prompt classified as internal; `docs/SECURITY.md` data classification policy |
| **LLM06 Excessive Agency** | HIGH | Agents have Bash/Write/Read tools; could invoke unintended actions | F4 decision rights per agent `.md`; human-in-the-loop for all state-changing actions (MISSION.md Q3); PFCA gate requires human sign-off |
| **LLM07 System Prompt Leakage** | MEDIUM | Users interacting with chat may probe system prompt boundaries | Instruct model to refuse system prompt disclosure; monitor for extraction patterns; do not embed secrets in system prompt |
| **LLM10 Unbounded Consumption** | MEDIUM | Long-running conversation agents, batch knowledge base processing | Per-request token budget; rate limiting on `/api/` endpoints; circuit breaker on LLM provider calls |
