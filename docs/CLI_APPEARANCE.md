## Звуки

Синтезированы скриптом (python, чистый stdlib) 2026-06-05: стерео, эффект объёма за счёт
задержки Хааса между каналами и эха с противоположной стороны. Палитра: done — мажорный
перелив C5-E5-G5, attention — два восходящих тона, agent — короткий «пузырёк», halt — низкий
двойной тон. Копии: `sounds/` здесь в jidoka.

## /stats — аналитика использования

`node scripts/cc-stats.mjs [--days N]` — читает историю `~/.claude/projects/**/*.jsonl`,
показывает: вывод токенов по дням (бары), топ проектов, доли моделей, итоги (сессии, ответы,
кэш). Только реальные числа из API, никаких оценок в долларах. ~1 сек на 7 дней истории.

## Строка статуса показывает

`🟢 jidoka eval 100% │ ⎇ main │ ✦ Opus 4.8 │ ctx ▰▰▰▰▱▱▱▱ 47% │ +120 −45 │ $1.23 │ ⚡max │ 5h ▰▰▰▱▱ 62%`

- здоровье jidoka (зелёный / жёлтый / красный HALT) — только в репо с фреймворком
- ветка git, модель
- прогресс-бар заполнения окна контекста (голубой < 60%, жёлтый < 85%, оранжевый дальше)
- добавленные/удалённые строки кода за сессию (синий/оранжевый — дальтоник-безопасно)
- стоимость сессии в долларах
- маркер усилия (только если max / xhigh / low)
- длительность сессии ⏱ (скрыта первую минуту)
- бары 5-часового и 7-дневного лимитов — появляются только когда израсходовано ≥ 50%
- СТРОКА 2 — «пульт миссии», промпт-якорь: `▶ 4m ◇ <последний промпт>`. Статусы: ▶ работаю
  (с таймером от момента промпта), ⏳ жду ответа (золотым), ✓ готово. Хук `session-state.mjs`
  слушает 4 события (UserPromptSubmit / PreToolUse / Stop / Notification) и ведёт
  `~/.claude/session-env/state-<session_id>.json`.
- СТРОКА 3 — что происходит сейчас: `⚙ <текущее действие> │ ✓ 2/5 ▸ <текущая задача плана>`.
  Действие берётся из PreToolUse (человеческое описание инструмента), прогресс плана — из
  `~/.claude/todos/<sid>-agent-<sid>.json`. Строка исчезает, когда показывать нечего.
- `refreshInterval: 5` в statusLine — таймер тикает каждые 5 секунд даже без событий.

- Тема + продолжение: короткий промпт («делай все три», «да», «продолжай» — меньше 5 слов и 40
  знаков) не затирает смысл. Хук хранит `topic` = последний содержательный запрос, строка
  показывает `◇ <тема> ❯ <короткая команда>`; заголовок вкладки тоже берёт тему. Добавлено
  2026-06-05 после жалобы владельца: «⏳ жду ответа ◇ делай все три — непонятно, что все три».
- Таймер меняет цвет по длительности: серый < 10 мин, янтарный 10-30, оранжевый ≥ 30.
- Звук-напоминание `nudge.wav` (тихий, два низких тона): если работа идёт 15+ минут — мягкий
  сигнал, повтор каждые следующие 15 минут, новый промпт сбрасывает отсчёт. Срабатывает из
  statusline (он тикает каждые 5 сек), отметка `nudgedAt` в state-файле защищает от дублей.
- Заголовок вкладки терминала: `session-state.mjs` пишет `▶/⏳/✓ <промпт до 40 зн.>` прямо в
  `/dev/tty` (OSC 0) — статус виден из других вкладок. Никогда не пишет в stdout (на
  UserPromptSubmit stdout попал бы в контекст модели).

Самотесты: `node scripts/statusline-jidoka.mjs --self-test` (42 проверки),
`node hooks/session-state.mjs --self-test` (17 проверок: переходы, описания, заголовки).
История: v1 «закреплённый промпт» (capture-prompt.mjs, удалён) → v2 «пульт миссии» →
v3 «таймер-светофор + напоминание + заголовок вкладки», всё 2026-06-05.

## Тема Jidoka Neon

База — `dark-daltonized` (цвета diff остаются дальтоник-безопасными, синий/оранжевый).
Принцип (выведен из двух итераций с владельцем): фон тёмно-зелёный, поэтому акценты —
НЕ зелёная семья (тонут), а яркие тёплые ИЛИ нейтральные светлые. Палитра: мятный бренд
`#2be8b0` (claude/борт промпта/success), почти белый `#eaf2ee` (план-режим, ide),
яркое золото `#ffca6e` (permission — должен ловить глаз), янтарь `#e0af68` (auto-accept,
warning), светло-серая рамка bash `#88a39b`, приглушённый текст — светлый нейтральный
серый `#93a8a1` (suggestion/subtle), inactive `#5b6e67`, remember бледно-мятный `#c9ead9`,
fastMode `#ffd866`, error `#ff7a93`.

История: v1 (2026-06-05 утро) — план/bash/remember из Tokyo Night (`#7aa2f7`, `#bb9af7`,
`#565f89`): фиолетовый выбивался на зелёном, сине-серый subtle нечитаем. v2 (вечер) —
всё в зелёную семью (`#4fd2e3`, `#2fae93`, `#6fa18c`): бирюзовое утонуло в бирюзовом фоне,
серо-зелёный subtle всё ещё тусклый. v3 (вечер, текущая) — принцип выше: тёплые + нейтральные
светлые. НЕ возвращать акценты в сине-фиолетовую или зелёную семью.

## Сквозные правила работы (2026-06-05, по запросу владельца)

1. **Чтение иерархии спек обязательно** перед фазами spec и build: `get-spec-context.mjs --feature <x>`,
   цепочка L0→L3. Записано: `HIERARCHICAL_SPEC_SYSTEM.md` §3 (v1.1.0), `COMMUNICATION_STYLE.md`,
   глобальный `~/.claude/CLAUDE.md`, скилл dev-pipeline.
2. **Коммуникация как полноценный сотрудник**: куда идём → план → «шаг N из M, делаю X ради Y» →
   итог с доказательством и бизнес-смыслом. Записано: `COMMUNICATION_STYLE.md` §Work narration,
   глобальный `~/.claude/CLAUDE.md`, скилл dev-pipeline.
3. **Коммит и пуш в main — всегда, без вопросов** (приказ владельца: «все в меин коммить и пуш!
   всегда»). В СОБСТВЕННЫХ репозиториях владельца: завершённая работа с зелёными тестами/гейтами
   коммитится и пушится так, чтобы попасть в `main` (fast-forward `git push origin <ветка>:main`
   или merge). Незакоммиченной готовой работы в конце захода не остаётся. Гейты pre-commit/pre-push
   обязательны, `--no-verify` запрещён. ВАЖНО: работа в установленной копии `~/.claude/jidoka` не
   под git — зеркалить в `~/claude-code-dev-framework` и коммитить там. ИСКЛЮЧЕНИЕ (сильнее этого
   правила): чужие/продакшен репо (Castells и т.п.) только для чтения, туда не пушить никогда.

## Разрешения без вопросов (2026-06-05, по прямому требованию владельца)

Все команды и инструменты выполняются БЕЗ подтверждения (владелец потребовал трижды;
финально: «баш команды делай все без моего запроса»). Эволюция решения: allow-правила
(`Bash(*)` + «голые» формы + Skill/Agent + `additionalDirectories`) убрали часть вопросов,
но встроенная защита самого Claude Code (команды, трогающие его хуки/настройки) спрашивала
всё равно. Финал: `permissions.defaultMode: "bypassPermissions"` +
`skipDangerousModePermissionPrompt: true`. Жёсткая защита остаётся на двух уровнях:
хук jidoka-guard (блокирует опасные паттерны механически) и обязанность ассистента САМОМУ
спросить перед внешними отправками, необратимыми удалениями, пушами во внешние репо,
секретами и продакшеном (записано в глобальном CLAUDE.md).

## Как откатить

- Тема: `/theme` → выбрать `dark-daltonized`
- Полноэкранный режим: убрать `env.CLAUDE_CODE_NO_FLICKER` из `settings.json`
- Звуки: убрать хуки `Stop` / `Notification` из `settings.json`
