# UX Flow: терминальная панель dev-пайплайна (TUI)

**Пользователь.** Нетехнический владелец. Не читает код. Смотрит: что идёт, что застряло, в каком терминале работа, и всё ли хорошо.

**Единственное действие пользователя.** Просмотр. Панель view-only. Единственные клавиши: q (выход), r (обновить вручную), стрелки влево/вправо (переключить проект).

---

## Приоритет внимания (сверху вниз)

1. HALT (красный баннер, всю ширину) — блокирует всё остальное
2. ЗАВИСЛО — если есть зависшие волны, секция идёт сразу после шапки
3. Активные волны на доске
4. Завершённые волны
5. Лента и терминалы

---

## Кодировка этапов: символ + метка (без опоры на цвет)

Дальтонник видит символ, не только цвет. Цвет добавляет смысл, но не несёт его один.

| Статус   | Символ | Цвет (ANSI) | Значение         |
|----------|--------|-------------|------------------|
| done     | ✓      | зелёный     | завершено        |
| running  | ▸      | жёлтый      | идёт сейчас      |
| stuck    | !      | красный     | зависло / нет хода |
| pending  | ○      | серый       | ещё не началось  |
| idle     | ◦      | тёмно-серый | пропущено / пусто |

Источник символов: ICON-объект из ui.html (✓ ▸ ✗ ○ ◦); `!` заменяет ✗ для "застряло" (ошибка без провала — отдельный случай).

Этапы доски — русские короткие названия, в том же порядке, что и pipeline в ui.html:

ПОИСК → СПЕК → ТЕСТЫ → СБОРКА → ГЕЙТ → ДЕБАГ → ПАМЯТЬ

## Состояние 1: НОРМАЛЬНОЕ (есть волны)

```
╔══════════════════════════════════════════════════════════════════════════════════╗
║  🦞 jidoka  🟢 GREEN  eval 94%          [mosco] [framework ⚙]      main        ║
║  3 волны в полёте · последнее обновление 14:32:07                               ║
╚══════════════════════════════════════════════════════════════════════════════════╝

─── ЗАВИСЛО (1) ──────────────────────────────────────────────────────────────────
  ! wave-auth-sso     ГЕЙТ      застыло 2ч 14м — ждёт ответа владельца
──────────────────────────────────────────────────────────────────────────────────

─── ДОСКА ────────────────────────────────────────────────────────────────────────
  ПОИСК      СПЕК       ТЕСТЫ      СБОРКА     ГЕЙТ       ДЕБАГ      ПАМЯТЬ
  ─────────  ─────────  ─────────  ─────────  ─────────  ─────────  ─────────
  ○ wave-ui  ▸ wave-db  ○ –        ✓ wave-x   ! wave-    ○ –        ✓ wave-y
             19%        –          100%        auth       –          100%
             [▓▓░░░░]              [▓▓▓▓▓▓]   sso        –          [▓▓▓▓▓▓]
  ○ wave-    –          –          –           25%        –          –
  tui-top                                     [▓░░░░░]
  0%
  [░░░░░░]

─── ТЕРМИНАЛЫ ────────────────────────────────────────────────────────────────────
  wave-db-refactor   →  tmux: jidoka / окно 2
  wave-auth-sso      →  tmux: jidoka / окно 3
  wave-tui-top       →  tmux: jidoka / окно 4

─── ЛЕНТА (последние действия) ───────────────────────────────────────────────────
  14:31  spec-architect    wave-db-refactor    написал спек модуля
  14:28  frontend-agent    wave-x              сборка завершена ✓
  14:22  gate-agent        wave-auth-sso       ГЕЙТ не прошёл — ждёт владельца !

──────────────────────────────────────────────────────────────────────────────────
  q выход · r обновить · ← → проект
```

Заметки к нормальному состоянию:
- Шапка: цветной индикатор (символ + слово) + eval% + имя ветки. Не только кружок, потому что дальтоники не различают красный/зелёный.
- Доска: прогресс-бар из символов `▓░`, 6 позиций = ~16% на символ. Рядом с баром — процент цифрой.
- Зависло: всегда первая секция после шапки, если есть хоть одна зависшая волна. Показывает сколько времени висит.
- Терминалы: простая таблица без технического жаргона — "tmux: имя сессии / окно N".
- Лента: время + агент + волна + коротко что. Не технические названия агентов — заменить на роли (spec-architect → "автор спека").

## Состояние 2: ANDON-HALT

```
╔══════════════════════════════════════════════════════════════════════════════════╗
║  🦞 jidoka  🔴 HALT                      [mosco] [framework ⚙]      main       ║
╚══════════════════════════════════════════════════════════════════════════════════╝

┌──────────────────────────────────────────────────────────────────────────────────┐
│  🔴 СТОП — пайплайн остановлен                                                  │
│                                                                                  │
│  Причина:  gate-constitutional · волна wave-auth-sso                             │
│  Когда:    сегодня 13:47                                                         │
│  Что делать: владелец должен дать команду resume                                 │
│             node .jidoka/scripts/andon-resume.mjs                                │
└──────────────────────────────────────────────────────────────────────────────────┘

─── ДОСКА (заморожена) ──────────────────────────────────────────────────────────
  ПОИСК    СПЕК     ТЕСТЫ    СБОРКА   ГЕЙТ     ДЕБАГ    ПАМЯТЬ
  ─────    ─────    ─────    ─────    ─────    ─────    ─────
  ◦        ◦        ◦        ◦        ! wave-  ◦        ◦
                                      auth-sso
                                      25%

  Все волны на паузе до снятия стопа.

──────────────────────────────────────────────────────────────────────────────────
  q выход · r обновить · ← → проект
```

Заметки:
- Баннер занимает всю ширину, рамка из символов `┌┐└┘─│`.
- Доска остаётся видна, но серая. Это показывает что именно заморожено.
- Команда resume написана дословно — владелец может передать её разработчику.
- "Нетехническое" объяснение ("что делать") обязательно.

```
╔══════════════════════════════════════════════════════════════════════════════════╗
║  🦞 jidoka  🟢 GREEN  eval 100%          [mosco] [framework ⚙]      main       ║
╚══════════════════════════════════════════════════════════════════════════════════╝

  Нет активных волн.

─── НЕДАВНО ЗАВЕРШИЛИСЬ ──────────────────────────────────────────────────────────
  ✓ wave-auth-basic      завершена вчера 18:02   100%
  ✓ wave-onboarding-v2   завершена 03.06  11:45   100%
  ✓ wave-perf-audit      завершена 01.06  09:20   100%

─── КАК ЗАПУСТИТЬ ВОЛНУ ──────────────────────────────────────────────────────────
  Скажи Claude: "запусти dev-pipeline для [название задачи]"
  или запусти вручную: node .jidoka/scripts/common-launcher.mjs
```

Заметки:

- Последние завершённые волны дают пользователю контекст, что система работала.
- Подсказка "как запустить" — в простых словах.

## Состояние 4: НЕСКОЛЬКО ПРОЕКТОВ (переключение)

```
╔══════════════════════════════════════════════════════════════════════════════════╗
║  🦞 jidoka  🟡 YELLOW  eval 82%    [mosco ●] [projectx] [framework ⚙]  main   ║
╚══════════════════════════════════════════════════════════════════════════════════╝

  Проект: mosco  (← → чтобы переключить)
  ...
```

Заметки:
- Активный проект выделен `●` и/или инвертом `[mosco ●]` — не только цветом.
- Клавиши ← → переключают. Показаны в подвале на всех состояниях.
- Health-индикатор меняется под каждый проект.

## Состояние 5: УЗКИЙ ТЕРМИНАЛ (<100 колонок)

При ширине < 100 колонок панель схлопывается так:

```
╔════════════════════════════════════════════════╗
║  🦞 jidoka  🟢  3 волны · main               ║
╚════════════════════════════════════════════════╝

! ЗАВИСЛО: wave-auth-sso (ГЕЙТ, 2ч 14м)

▸ wave-db-refactor    СПЕК    19%  [▓▓░░░░]
✓ wave-x              СБОРКА  100%
! wave-auth-sso       ГЕЙТ    25%  [▓░░░░░]
○ wave-tui-top        ПОИСК   0%

ТЕРМИНАЛЫ
  wave-db  → tmux jidoka/2
  wave-sso → tmux jidoka/3

ЛЕНТА
  14:31 gate-agent wave-auth-sso ! гейт не прошёл

────────────────────────────────────────────────
q выход · r обновить · ←→ проект
```

Что схлопывается первым (по порядку):
1. Доска колонок → линейный список волн с символом+этапом+%
2. Прогресс-бар сужается до 6 символов (или убирается при <80 колонок)
3. Детали терминалов — сессия без пояснений
4. Шапка теряет eval%, оставляет только значок health и счётчик волн
5. Подвал сжимается: `q · r · ←→`

## Состояние 6: НЕ-TTY (вывод в pipe / redirect)

Когда панель запускают без терминала (например, `tui-panel | grep HALT`), вместо рисования рамок выводится плоский снимок:

```
jidoka-snapshot 2026-06-05T14:32:07Z
health GREEN eval=94% halt=false branch=main
waves 3 inflight

WAVE wave-db-refactor stage=СПЕК progress=19% status=running terminal=tmux:jidoka/2
WAVE wave-x stage=СБОРКА progress=100% status=done
WAVE wave-auth-sso stage=ГЕЙТ progress=25% status=stuck terminal=tmux:jidoka/3
WAVE wave-tui-top stage=ПОИСК progress=0% status=pending terminal=tmux:jidoka/4

STUCK wave-auth-sso gate stuck_since=2h14m

ACTIVITY 14:31 gate-agent wave-auth-sso action=fail label="гейт не прошёл"
ACTIVITY 14:28 frontend-agent wave-x action=done
```

Формат: одна запись на строку, ключ=значение без пробелов, machine-readable. Не нужно парсить рамки.

## Информационная архитектура: что первично, что вторично

| Блок | Приоритет | Условие показа |
|------|-----------|----------------|
| ANDON-HALT баннер | 1 | только при halt=true |
| ЗАВИСЛО (stuck-секция) | 2 | только если есть волны со статусом stuck |
| Шапка (health + проекты + ветка) | 3 | всегда |
| Доска колонок | 4 | всегда (пустая если нет волн) |
| Терминалы | 5 | только если хотя бы одна волна имеет terminal |
| Лента действий | 6 | всегда, но до 5 строк |
| Подвал с клавишами | 7 | всегда |
| Недавно завершённые | только в пустом состоянии |

Убрано по сравнению с ui.html: KPI-кольца, уроки мета-леджера, timeline коммитов, раздел Production. Эти данные нужны разработчику, не владельцу. В TUI они создадут шум.

## Friction audit

| Шаг | Усилие | Решение |
|-----|--------|---------|
| Найти зависшую волну | высокое без секции ЗАВИСЛО | вынести наверх отдельно, не прятать в доску |
| Понять что значит "ГЕЙТ" | среднее | рядом — сколько времени висит и короткое объяснение |
| Понять в каком терминале работа | высокое без секции | явная секция ТЕРМИНАЛЫ, простой формат |
| Понять что делать при HALT | высокое | баннер содержит команду resume дословно |
| Переключить проект | низкое | стрелки, всегда видны в подвале |

## Взаимодействие с данными

Панель читает те же источники, что ui.html и statusline-jidoka.mjs:
- `docs/audits/andon-halt.json` или `.jidoka/andon-halt.json` — признак halt
- `.jidoka/_baseline.json` — eval%
- API `/api/data?project=` — волны, board, activity (если сервер запущен)
- Fallback без сервера — читает run-state файлы напрямую

Обновление: автоматически раз в 5 секунд. Клавиша `r` — немедленно. Без мигания всего экрана: обновляется только содержимое, рамки остаются.

## Что нужно от design-system-architect

Новые примитивы не нужны. Все символы (✓ ▸ ! ○ ◦ ▓ ░) и ANSI-цвета (зелёный/жёлтый/красный/серый) уже есть в statusline-jidoka.mjs и ui.html. Рамки из псевдографики `╔╗╚╝═║┌┐└┘─│` — стандарт терминала.

Единственное уточнение: зафиксировать точные ANSI-коды для каждого уровня health, чтобы не расходиться с statusline. statusline уже использует `\x1b[32m` (зелёный) и `\x1b[31m` (красный) — TUI должен использовать те же коды.
